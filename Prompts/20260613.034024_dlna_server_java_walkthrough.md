# User Prompt

> Create a complete Java CLI application that functions as a DLNA/UPnP AV Media Server.
> The application must expose a user-specified directory as a DLNA media library.
> DLNA-compatible devices (Smart TVs, VLC, Kodi, Windows Media Player, Android DLNA clients, etc.)
> should automatically discover the server on the local network.

## Key Fixes & Decisions Made

### 1. Library Migration: Cling → jUPnP

**Problem:** The original Cling (`org.fourthline.cling`) library is abandoned and its Maven repository (`4thline.org`) is offline — builds fail unconditionally.

**Resolution:** Migrated to `org.jupnp:org.jupnp:3.0.4` on Maven Central — the active community fork of Cling, used in production by Eclipse openHAB.

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.jupnp</groupId>
    <artifactId>org.jupnp</artifactId>
    <version>3.0.4</version>
</dependency>
<dependency>
    <groupId>org.jupnp</groupId>
    <artifactId>org.jupnp.support</artifactId>
    <version>3.0.4</version>
</dependency>
```

---

### 2. API Migration: jUPnP 3.x Breaking Changes from Cling

| Old Cling API | jUPnP 3.0.4 Replacement |
|---|---|
| `import org.fourthline.cling.*` | `import org.jupnp.*` |
| `DIDParser` | `DIDLParser` (same package: `org.jupnp.support.contentdirectory`) |
| `ErrorCode` from `org.jupnp.support.model` | `ErrorCode` from **`org.jupnp.model.types.ErrorCode`** |
| `StorageFolder(id, parentId, title, childCount)` | `StorageFolder(id, parentId, title, creator, Integer childCount, Long storageUsed)` |
| `new Photo(id, parentId, title, album, res)` | Setter-based: `ph.setId(); ph.setParentID(); ph.addResource(res)` |

---

### 3. ConnectionManagerService Duplicate Class Error

**Problem:** Our class was named `ConnectionManagerService`, and it also imported `org.jupnp.support.connectionmanager.ConnectionManagerService` — Java rejected this as a duplicate class definition in the same compilation unit.

**Fix:** Removed the import and extended via fully-qualified name:
```java
public class ConnectionManagerService
        extends org.jupnp.support.connectionmanager.ConnectionManagerService {
    // no import needed
}
```

---

### 4. NullPointerException: `getRegistry()` is null

**Problem:**
```
NullPointerException: Cannot invoke "Registry.addDevice(...)"
because the return value of "UpnpService.getRegistry()" is null
```

**Root Cause:** In jUPnP 3.x, unlike old Cling, `UpnpServiceImpl` does **not** auto-start in its constructor. `getRegistry()` returns null until `startup()` is called.

**Fix:**
```java
upnpService = new UpnpServiceImpl(new DefaultUpnpServiceConfiguration());
upnpService.startup(); // ← Must be called explicitly in jUPnP 3.x
```

---

### 5. NullPointerException: `getConfiguration()` is null inside `startup()`

**Problem:**
```
NullPointerException: Cannot invoke "UpnpServiceConfiguration.getClass()"
because the return value of "UpnpServiceImpl.getConfiguration()" is null
at UpnpServiceImpl.startup(UpnpServiceImpl.java:256)
```

**Root Cause:** `UpnpServiceImpl()` (no-arg constructor) is an **OSGi-only** constructor. In OSGi, the configuration is injected via `@Reference`. When run standalone, `getConfiguration()` returns null.

**Fix:** Pass `DefaultUpnpServiceConfiguration` explicitly:
```java
// WRONG (OSGi-only):
upnpService = new UpnpServiceImpl();

// CORRECT (standalone Java):
upnpService = new UpnpServiceImpl(new DefaultUpnpServiceConfiguration());
```

`DefaultUpnpServiceConfiguration` also requires Jetty as a transport — add to `pom.xml`:
```xml
<dependency>
    <groupId>org.eclipse.jetty</groupId>
    <artifactId>jetty-server</artifactId>
    <version>9.4.56.v20240826</version>
</dependency>
<dependency>
    <groupId>org.eclipse.jetty</groupId>
    <artifactId>jetty-client</artifactId>
    <version>9.4.56.v20240826</version>
</dependency>
<dependency>
    <groupId>javax.servlet</groupId>
    <artifactId>javax.servlet-api</artifactId>
    <version>3.1.0</version>
</dependency>
```

## Functional Features

- ✅ **SSDP Discovery** — advertises as `upnp:rootdevice` + `urn:schemas-upnp-org:device:MediaServer:1`
- ✅ **ContentDirectory service** — responds to UPnP `Browse(BrowseDirectChildren)` and `Browse(BrowseMetadata)` with DIDL-Lite XML
- ✅ **ConnectionManager service** — advertises supported protocol infos (MP4, MKV, AVI, MP3, FLAC, WAV, JPEG, PNG)
- ✅ **HTTP Media Streaming** — JDK `HttpServer` with `Range` header support (206 Partial Content) for seekable playback
- ✅ **Live File Watching** — `NIO WatchService` re-scans on file system changes without restart
- ✅ **Cross-platform** — works on Windows, macOS, Linux

## Cling vs jUPnP Comparison

| Factor | Cling | jUPnP |
|---|---|---|
| Last release | ~2016 | 2024 (3.0.4) |
| Maven repo | ❌ Offline | ✅ Maven Central |
| Java 21+ | ❌ Broken | ✅ Supported |
| Maintenance | ❌ Abandoned | ✅ Active (openHAB) |

**Verdict:** Always use `org.jupnp` for any new or ongoing DLNA/UPnP Java project.

## Build & Run

```bash
# Build fat JAR
mvn clean package

# Run
java -jar target/dlna-server-1.0.0.jar --media-dir "C:\Users\YourName\Videos"
```

The server will:
1. Scan the media directory recursively
2. Start the HTTP streaming server on a random port
3. Register the UPnP device and broadcast SSDP presence
4. Smart TVs, VLC, Kodi, and other DLNA clients will auto-discover it within seconds
