# Implementation: DLNA Profile Auto-Discovery Cycler

## Summary

Since we cannot know in advance which DLNA `protocolInfo` the LG 47LA6200 will accept for a given MP4 file, we implement a **self-discovering cycler** that tries each candidate profile automatically, one per click, without restarting the server.

## How It Works

| Event | Action |
|-------|--------|
| Server start | Load saved winner from `dlna_profile_winner.log` (if exists), set as starting profile |
| User clicks MP4 on TV (BROWSE METADATA) | Cycler advances to next candidate; DIDL-Lite is returned with that profile |
| TV likes the profile → sends HTTP GET | `notifySuccess()` logs the winner to `dlna_profile_winner.log` |
| Next server start | Winner restored immediately as candidate #1 — working on first click |

## Candidate Profiles (14 total, in priority order)

1. No PN — FLAGS=01700000 *(broadest fallback)*
2. `AVC_MP4_MP_HD_1080i_AAC` — FLAGS=01700000
3. `AVC_MP4_MP_HD_720p_AAC` — FLAGS=01700000
4. `AVC_MP4_BL_L3L_SD_AAC` — FLAGS=01700000
5. `AVC_MP4_MP_SD_AAC_MULT5` — FLAGS=01700000
6. `AVC_MP4_HP_HD_AAC` — FLAGS=01700000
7. `AVC_MP4_MP_SD_MPEG1_L3` — FLAGS=01700000
8. No PN — FLAGS=01500000
9. `AVC_MP4_MP_HD_1080i_AAC` — FLAGS=01500000
10. `AVC_MP4_MP_HD_720p_AAC` — FLAGS=01500000
11. `AVC_MP4_BL_L3L_SD_AAC` — FLAGS=01500000
12. No PN — OP=00, FLAGS=01700000
13. `AVC_MP4_MP_HD_1080i_AAC` — OP=00, FLAGS=01700000
14. Bare wildcard `*`

## Files Changed

| File | Change |
|------|--------|
| **NEW** `DlnaProfileCycler.java` | Core cycler class with 14 candidates, advance/success/log/restore logic |
| `ContentDirectoryService.java` | Uses cycler in `buildItem()`: advance on BROWSE METADATA (click), read-only on BROWSE DIRECT_CHILDREN (folder listing) |
| `HttpStreamingServer.java` | Calls `cycler.notifySuccess()` on first MP4 GET; mirrors cycler candidate in `contentFeatures.dlna.org` header |
| `DlnaMediaServer.java` | Passes cycler through to `ContentDirectoryService` |
| `Application.java` | Creates cycler singleton, calls `loadWinner()` on startup |

## Winner Log Format

`dlna_profile_winner.log` (in the server's working directory):
```
2026-06-17 05:30:12 | WINNER #2 | candidate=[DLNA.ORG_PN=AVC_MP4_MP_HD_1080i_AAC;...] | protocolInfo=[http-get:*:video/mp4:...]
```

## Notes

- After finding the winner, you can hard-code that specific candidate in `DlnaProfileCycler.CANDIDATES` at index 0 so it is always tried first.
- If none of the 14 profiles work, the file's internal codec (H.265, DTS audio, etc.) is the problem and re-encoding is required.
